# Color System

A **semantic, accessible color palette** designed around surfaces, content (text), accents, and feedback. All colors ensure WCAG AAA contrast compliance when paired correctly.

---

- **Inverse**: High-contrast overlays or accent backgrounds (rare, strategic use).

## 📝 Content (Text)

Text colors designed for readability and semantic meaning. Always respect WCAG AAA contrast ratios.

<ColorPalette>
  <ColorItem
    title="Primary (text-content-primary)"
    subtitle="Main text — guaranteed WCAG AAA on surfaces"
    colors={{
      Light: "#000000",
      Dark: "#ffffff",
    }}
  />
  <ColorItem
    title="Secondary (text-content-secondary)"
    subtitle="Supporting text — labels, captions, secondary info"
    colors={{
      Light: "#4a4a4a",
      Dark: "#d4d4d4",
    }}
  />
  <ColorItem
    title="Tertiary (text-content-tertiary)"
    subtitle="De-emphasized text — hints, disabled state, metadata (use sparingly)"
    colors={{
      Light: "#737373",
      Dark: "#737373",
    }}
  />
  <ColorItem
    title="Inverse (text-content-inverse)"
    subtitle="Text on inverted surfaces — contrast text"
    colors={{
      Light: "#ffffff",
      Dark: "#000000",
    }}
  />
</ColorPalette>

### Usage & A11y Guidance

- **Primary**: Use for all body text, headings, and interactive labels. WCAG AAA compliant at all sizes.
- **Secondary**: Use for supporting text, form labels, and secondary UI. Maintains strong contrast.
- **Tertiary**: Reserve for non-essential UI (timestamps, metadata, disabled buttons). **Never** for main body text or primary CTAs.
- **Inverse**: Use strictly on inverted backgrounds (dark overlays, inverse surfaces).

> **A11y Best Practice**: Always verify contrast ratios in your actual implementation. Use tools like WebAIM's Contrast Checker to validate against WCAG AAA (7:1 ratio for body, 4.5:1 for large text).

## 🎲 Accessibility Checklist

- ✅ All `text-content-*` colors on default surfaces meet WCAG AAA (7:1 ratio).
- ✅ `text-content-secondary` maintains strong contrast for UI labels and secondary text.
- ✅ `text-content-tertiary` reserved for non-essential UI; **never** as primary content.
- ✅ Never rely on color alone for information (pair with icons, patterns, or text).
- ✅ Test dark mode contrasts in your actual implementation (CSS variables are scoped).
